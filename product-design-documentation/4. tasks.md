### **Epic 1: Backend Infrastructure & Security**
*Focus: Setting up the Single Source of Truth (SSOT) and securing the connection.*

| Task ID | Component | Task Description |
| :--- | :--- | :--- |
| **BE-1.1** | Database | **Initialize MySQL Schema:** Create `families`, `users`, and `chores` tables with defined FK constraints and UUID primary keys. |
| **BE-1.2** | API | **Auth Middleware:** Implement a PHP script to validate the `X-CHORES-TOKEN` header on every incoming request. |
| **BE-1.3** | API | **Family/User Creation:** Develop `POST /api/family/create` and `POST /api/users/add` endpoints to register the initial household data. |
| **BE-1.4** | API | **Chore Fetching:** Implement `GET /chores.php` with a filter to exclude records where `deleted_at` is not null. |
| **BE-1.5** | API | **Claim Validation:** Add server-side logic to reject claim actions with a **409 Conflict** if the chore is already assigned to another user. |
| **BE-1.6** | API | **Soft-Delete Logic:** Update the delete endpoint to set the `deleted_at` timestamp rather than purging the record. |
